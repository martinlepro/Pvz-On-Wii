# Wii Disc Components (ISO / WBFS)

When you run `make iso`, **Wiimms ISO Tools (WIT)** assembles a bootable Wii disc image from
the staged file tree under `disc/staging/`. You do **not** need to hand-edit ticket, TMD, or
partition tables for normal homebrew builds — WIT generates them during the pack step.

## Source layout (what you maintain)

| Path | Purpose |
|------|---------|
| `disc/staging/sys/main.dol` | Main executable (copied from `pvz_wii.dol` at build time) |
| `disc/staging/files/` | Optional game data (mirrors runtime `assets/` if present) |
| `disc/assets/opening.bnr` | Optional disc banner shown in Dolphin / Wii System Menu |
| `disc/meta/disc.info` | Title, 4-char game ID, region hint for WIT |

## Generated by WIT (do not commit)

WIT writes these **inside** the `.iso` / `.wbfs` container when packing:

| Component | Role |
|-----------|------|
| **Boot partition / boot.bin** | IPL2 bootstrap that loads the disc menu executable |
| **Ticket** | Encrypted title key + rights for the disc title |
| **TMD (Title Metadata)** | Content hashes, IOS version, access control |
| **Cert chain** | Nintendo certificate chain (homebrew uses standard patterns) |
| **FST (File System Table)** | Directory index for `sys/` and `files/` |
| **DOL partition** | Wraps `sys/main.dol` as the disc's main binary |

Homebrew images use a **custom 4-character ID** (see `disc.info`, default `PVZW`) so they
do not collide with retail titles.

## Optional: custom `opening.bnr`

1. Run `python scripts/generate_opening_bnr.py` to create a minimal banner, **or**
2. Build one with [CustomizeMii](https://github.com/IvanSanchez-CustomMii/CustomizeMii) and save it as `disc/assets/opening.bnr`.

If `opening.bnr` is missing, WIT still produces a bootable image; only the menu banner may be blank.

## Tooling reference

- [Wiimms ISO Tools (WIT)](https://wit.wiimm.de/) — primary packer used by this project
- [WiiBrew: WIT](https://wiibrew.org/wiki/Wiimms_ISO_Tools) — command overview
- Dolphin: **File → Open** → select `output/disc/pvz_wii.iso` or `pvz_wii.wbfs`
